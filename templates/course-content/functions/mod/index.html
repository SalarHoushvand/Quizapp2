Functions (mod)

Modular arithmetic is a common thread running through the topics of discrete structures. At this juncture, we use modular arithmetic to tie together the concepts around functions.

When an integer \(x\) in is divided by a nonnegative integer \(n,\) then there are unique integers \(q\) and \(r\) so that \(x = qn + r\) where \(q \geq 0\) and \(0 \leq r < n.\) The uniqueness and existence of such a pair of integers \(q\) and \(r\) is the result of the Divivion Algorithm. The integers are then partitioned into \(n\) non-overlapping sets called equivalence classes \([0],[1],\ldots,[n-1]\) that form a partition of \(\mathbb{Z}.\)

But what is even more beautiful is the mapping from \(Z\) to \(Z_n\) where \(n\) is a positive integer. The map that takes \(x\) in \(Z\) to the equivalence class element \([x]\) of \(Z_n\) where \(x = qn + r\) where \(q \in \mathbb{Z}^+ \cup \{0\}\) and \(0 \leq r < n\) so that the set \([x] = \{z \vert z \in \mathbb{Z} \text{ and } z = q_1n+r, \text{ with } q_1 \in mathbb{Z} \cup \{0\}\}\).

When \(n = 2,\) the map takes \(7 \longrightarrow [7]\) However, there is some flexibility in referring to the elements of \(Z_2\) since \([1] = \{ \ldots, -1, 1, 3, 5, 7 \ldots \}\) and \([-1] = [1] = [-3] = \cdots = [7] = \cdots\), there are an infinte number of ways for us to refer to the same element in \(Z_2.\) The same is true for the elements equivalent to \([0] \in Z_2\) such as \([2]\) and \([-2]\).

The set \(Z_n\) can have no repeated elements by definiton, and the convention is to only use the elements \([0], [1], \ldots, [n-1]\) when working with the set and operations on the set.

In general Modular Arithmetic is defined on elements (equivalence classes) \([x]\) and \([y]\) in \(Z_n\) with

\[[x] + [y] = [z], where (x + y) \equiv z \pmod{n}.\]

Example 1 In \(Z_7\), \([1] = \{\ldots, -6, 1, 8, \ldots\}\) and \([16] = \{ \ldots, -5, 2 , 9, 16, \ldots \}\) the result of adding \([1] + [3] = [4]\) in \(Z_2,\) which is reassuring to many. However, when adding \([3]\) and \([6]\) in $$Z_2, we have

\([3] + [6] = [2]\) since \((3 + 6) \equiv 9 \equiv 2 \pmod{2}.\)


To add 3 and 6 in \(Z_7,\) you can start at 6 and rotate clockwise 3 units to the 2 on the circle.

Set union and modular arithmetic are very different operations in \(Z_2\). Consider the union of \([0]\) and \([1]\) in \(Z_2.\)

\[[0] \cup [1] = Z\]

and modular arithmetic on the elements (equivalence classes) of \(Z_2\) gives us:

\([0] + [1] = [x]\) where \(x \equiv (0 + 1) \pmod{2}\)

Consider the functions \(f\) and \(g\) on the set \(D = \{0, 1, 2\}\) with range set \(\{0, 1, 2\}\) where addition on \(R\) is addition mod 3.
\(f(x) = \begin{cases} 0 & x = 0 \text{ or } x = 1\\ 2 & x = 2\\ \end{cases}\) and \(g(x) = \begin{cases} 1 & x = 0 \text{ or } x = 1\\ 0 & x = 2\\ \end{cases}\)

Then \((D, R, f+g)\) is in the set \(N\) of all functions from \(\{0, 1, 2\}\) to itself and \((f + g)(x) = f(x) +_{R} g(x) = \begin{cases} 1 & x = 0 \text{ or } x = 1\\ 2 & x = 2\\ \end{cases}\)
